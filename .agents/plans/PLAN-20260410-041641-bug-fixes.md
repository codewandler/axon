# Bug Fix Plan — 2026-04-10

Three bugs surfaced during the v0.5.0 demo session.

---

## Bug inventory

| # | Bug | Root cause | Files |
|---|-----|-----------|-------|
| B1 | `NOT CONTAINS ANY/ALL` rejected by parser | Grammar only defines `NOT CONTAINS (list)`, not the `ANY`/`ALL` variants | `aql/parser.go`, `aql/ast.go`, `adapters/sqlite/aql.go` |
| B2 | `axon impact` never shows importing packages | `extractRefPkgPath` returns a filesystem path; lookup queries `data.import_path` (a Go import path) — the two never match | `cmd/axon/impact.go` |
| B3 | Variable-length path `SELECT *` crashes with "invalid character 'V'" | `SELECT n_end.*` returns columns in DDL order; the `isStar` scan branch expects them in a different order — causing a generation ID to be parsed as labels JSON | `adapters/sqlite/aql.go` |

## Root-cause detail

### B1 — `NOT CONTAINS ANY/ALL`

The parser grammar (participle struct tags) for label expressions is:

```go
// existing
NotContains bool     `parser:"  | @('NOT' 'CONTAINS')"`
LabelList1  []string `parser:"    '(' @String (',' @String)* ')'"`

ContainsAny bool     `parser:"  | 'CONTAINS' @'ANY'"`
LabelList2  []string `parser:"    '(' @String (',' @String)* ')'"`

ContainsAll bool     `parser:"  | 'CONTAINS' @'ALL'"`
LabelList3  []string `parser:"    '(' @String (',' @String)* ')'"`
```

`CONTAINS ANY/ALL` exist; the `NOT` variants do not. A user trying
`labels NOT CONTAINS ANY ('x')` gets a parse error pointing at `ANY`.

**Semantics to add:**
- `NOT CONTAINS ANY (...)` — none of the listed labels present (same as `NOT CONTAINS`)
- `NOT CONTAINS ALL (...)` — not all of the listed labels present (at least one missing)

### B2 — `axon impact` import path mismatch

`go:ref` nodes carry:
```json
{ "position": { "file": "/home/timo/projects/axon/aql/builder.go" },
  "target_pkg": "github.com/codewandler/axon/aql" }
```

`extractRefPkgPath` calls `filepath.Dir(position.file)` →
`/home/timo/projects/axon/aql` (filesystem path).

Step 4 then queries:
```sql
SELECT * FROM nodes WHERE type = 'go:package'
  AND data.import_path = '/home/timo/projects/axon/aql'
```

But `data.import_path` stores `github.com/codewandler/axon/aql`. No match.

`go:package` nodes have a `data.dir` field that IS the filesystem directory.
The bridge is: `dir → data.dir → go:package → data.import_path`.

**Fix:** build a map of `dir → import_path` from all `go:package` nodes once,
then use it to translate filesystem directories into import paths for the
impact-graph lookup. Also: `target_pkg` (the referenced package) was being
used as a stand-in for the referencing package — fix `extractRefPkgPath` to
only use `position.file`.

### B3 — Variable-length `SELECT *` column order mismatch

The SQLite `nodes` table DDL column order:
```
id, type, uri, key, data, generation, root, created_at, updated_at, name, labels
```

`SELECT n_end.*` returns columns in that DDL order.

`executeNodeQuery`'s `isStar` branch scans expecting:
```
id, type, uri, key, name, labels, data, generation, root, created_at, updated_at
```

The mismatch puts `data` (JSON) into `name`, `generation` (a nanoid) into
`labelsStr`, and `json.Unmarshal(nanoid)` fails — "invalid character 'X'
looking for beginning of value" where X is the first char of the nanoid.

**Fix:** replace `SELECT n_end.*` / `SELECT n_start.*` with an explicit column
list that matches the scan order:
```sql
SELECT n.id, n.type, n.uri, n.key, n.name, n.labels, n.data,
       n.generation, n.root, n.created_at, n.updated_at
```

### Task 3 — Handle new operators in the SQLite AQL compiler

**File:** `adapters/sqlite/aql.go`
**Estimated time:** 5 min

Find the `compileLabelExpr` function (or wherever `OpNotContains`,
`OpContainsAny`, `OpContainsAll` are handled). Add cases for the new ops.

`NOT CONTAINS ANY (a, b)` is logically identical to `NOT CONTAINS (a, b)`:
the labels JSON array does not contain any of the listed values. Reuse
the same SQL as `OpNotContains`.

`NOT CONTAINS ALL (a, b)` means "not all of {a,b} are present" — i.e., at
least one is absent. SQL:
```sql
NOT (labels LIKE '%"a"%' AND labels LIKE '%"b"%')
```
Or, using the same JSON approach as `OpContainsAll` but negated:
```sql
NOT (json_each_match_all...)
```
The simplest correct implementation uses a NOT around the ALL logic:

```go
case OpNotContainsAll:
    // NOT (label1 present AND label2 present AND ...)
    // = same SQL as OpContainsAll wrapped in NOT (...)
    inner := compileContainsAll(labels)  // whatever compileContainsAll produces
    return "NOT (" + inner + ")", args, nil
```

If there's no `compileContainsAll` helper, inline it and wrap with NOT.

### Task 5 — Fix `axon impact`: map filesystem dirs to import paths

**File:** `cmd/axon/impact.go`
**Estimated time:** 10 min

**Step A — build a dir→importPath lookup before processing refs:**

```go
// Load all go:package nodes and build a dir→importPath map.
pkgQuery, _ := aql.Parse(`SELECT * FROM nodes WHERE type = 'go:package'`)
pkgResult, err := storage.Query(ctx, pkgQuery)
if err != nil {
    return fmt.Errorf("loading packages: %w", err)
}
dirToImportPath := make(map[string]string) // filesystem dir → Go import path
for _, pkg := range pkgResult.Nodes {
    d, ok := pkg.Data.(map[string]interface{})
    if !ok {
        continue
    }
    dir, _ := d["dir"].(string)
    ip,  _ := d["import_path"].(string)
    if dir != "" && ip != "" {
        dirToImportPath[dir] = ip
    }
}
```

**Step B — fix `extractRefPkgPath` to return only the directory:**

The function already does this — it calls `filepath.Dir(position.file)`.
The problem is downstream. Leave `extractRefPkgPath` as-is; it correctly
returns the filesystem directory of the referencing file.

**Step C — translate dirs to import paths in the grouping loop:**

After building `refsByPkg` (keyed by filesystem dir), translate keys:

```go
// Re-key refsByPkg by import path (where known).
importPathGroups := make(map[string]*refGroup)
for fsDir, g := range refsByPkg {
    ip, ok := dirToImportPath[fsDir]
    if ok {
        g.pkgPath = ip
        importPathGroups[ip] = g
    } else {
        // Keep the raw dir as a fallback key.
        importPathGroups[fsDir] = g
    }
}
refsByPkg = importPathGroups
```

**Step D — fix the package lookup in Step 4 (import graph):**

With `pkgPath` now being a proper Go import path, replace:
```go
// OLD — queries by import_path (a Go path) using filesystem value — broken
pkgQuery, _ := aql.Parse(fmt.Sprintf(
    `SELECT * FROM nodes WHERE type = 'go:package' AND data.import_path = '%s' LIMIT 1`,
    escapeSQL(pkgPath)))
```
with a direct map lookup using the already-loaded `pkgResult.Nodes`:

```go
// Build importPath→nodeID map once (before the loop).
importPathToID := make(map[string]string)
for _, pkg := range pkgResult.Nodes {
    d, ok := pkg.Data.(map[string]interface{})
    if !ok { continue }
    ip, _ := d["import_path"].(string)
    if ip != "" {
        importPathToID[ip] = pkg.ID
    }
}
```

Then in Step 4:
```go
pkgNodeID, ok := importPathToID[pkgPath]
if !ok {
    continue
}
importerEdges, err := storage.GetEdgesTo(ctx, pkgNodeID)
```

No extra queries needed — all packages were already loaded in Step A.

### Task 7 — Add a regression test for variable-length `SELECT *`

**File:** `adapters/sqlite/aql_test.go`

```go
func TestPattern_VariableLength_SelectStar(t *testing.T) {
    s, err := New(":memory:")
    if err != nil {
        t.Fatalf("sqlite.New: %v", err)
    }
    defer s.Close()
    ctx := context.Background()

    root   := graph.NewNode("fs:dir").WithURI("file:///r").WithName("root")
    child  := graph.NewNode("fs:file").WithURI("file:///r/f.go").WithName("f.go")
    s.PutNode(ctx, root)
    s.PutNode(ctx, child)
    s.PutEdge(ctx, graph.NewEdge("contains", root.ID, child.ID))
    s.Flush(ctx)

    // SELECT * must not crash — previously broke with "invalid character"
    // because n_end.* returns columns in DDL order, not scan order.
    q, err := aql.Parse(`SELECT * FROM (r:fs:dir)-[:contains*1..1]->(f:fs:file)`)
    if err != nil {
        t.Fatalf("parse: %v", err)
    }
    result, err := s.Query(ctx, q)
    if err != nil {
        t.Fatalf("Query: %v", err)   // was crashing here
    }
    if len(result.Nodes) == 0 {
        t.Fatal("expected at least one node")
    }
    if result.Nodes[0].Name != "f.go" {
        t.Errorf("expected f.go, got %q", result.Nodes[0].Name)
    }
}
```
